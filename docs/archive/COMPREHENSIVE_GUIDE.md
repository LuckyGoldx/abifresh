## 📊 System Architecture

### Frontend Architecture
```
App (Next.js)
├── Pages
│   ├── /login - Unified login
│   ├── /admin/dashboard - Admin dashboard
│   ├── /sales/dashboard - Sales dashboard
│   ├── /staff/dashboard - Staff dashboard
│   └── /offline - Offline fallback
├── Components
│   ├── Header - Navigation
│   ├── Sidebar - Menu
│   └── ChatBot - AI assistant (future)
├── Lib
│   ├── api.ts - HTTP client with JWT
│   └── supabase.ts - Database client
├── Store
│   ├── auth.ts - Auth state
│   └── theme.ts - Theme state
└── Styles
    ├── globals.css - Global styles
    └── tailwind.config.js - Tailwind config
```

### Backend Architecture
```
Express Server (Node.js)
├── Middleware
│   ├── auth.ts - JWT validation
│   └── cors - CORS configuration
├── Routes
│   ├── /api/auth - Authentication
│   ├── /api/sales - Sales transactions
│   ├── /api/inventory - Inventory management
│   ├── /api/admin - Admin operations
│   ├── /api/staff - Staff operations
│   └── /api/chat - AI chatbot (future)
├── Services
│   ├── auth.service.ts - User management
│   ├── sales.service.ts - Sales logic
│   ├── inventory.service.ts - Inventory logic
│   ├── admin.service.ts - Admin operations
│   └── chat.service.ts - AI logic (future)
└── Config
    └── supabase.ts - Database connection
```

### Database Architecture
```
Supabase (PostgreSQL)
├── Authentication
│   └── auth.users (Supabase built-in)
├── Core Tables
│   ├── users - User accounts & roles
│   ├── items - Product catalog
│   └── inventory_* - Stock levels
├── Business Logic
│   ├── sales - Transactions
│   ├── daily_sales_summary - Daily totals
│   ├── posted_items - Staff postings
│   └── staff_payments - Payment requests
├── Management
│   ├── staff_commissions - Commission rates
│   ├── expenses - Staff expenses
│   ├── notifications - System notifications
│   └── activity_logs - Audit trail
└── Security
    └── RLS Policies - Row-level access control
```

## 🔌 API Overview

### Authentication (3 endpoints)
- `POST /api/auth/register` - Create account
- `POST /api/auth/login` - Get JWT token
- `GET /api/auth/me` - Get profile

### Sales (4 endpoints)
- `GET /api/sales/available-items` - List items
- `POST /api/sales/record` - Record transaction
- `POST /api/sales/post-items` - Post to staff
- `GET /api/sales/dashboard` - Sales stats

### Inventory (7 endpoints)
- `GET /api/inventory/main-store` - View main store
- `GET /api/inventory/active-store` - View active store
- `POST /api/inventory/add` - Add items
- `PUT /api/inventory/edit/:id` - Edit item
- `DELETE /api/inventory/delete/:id` - Delete item
- `POST /api/inventory/move` - Move between stores
- `GET /api/inventory/summary` - Summary stats

### Admin (8 endpoints)
- `GET /api/admin/staff` - List staff
- `POST /api/admin/staff` - Create staff
- `GET /api/admin/commissions` - View commissions
- `PUT /api/admin/commissions` - Set commission
- `GET /api/admin/payments` - Pending payments
- `PUT /api/admin/payments/:id/approve` - Approve
- `PUT /api/admin/payments/:id/reject` - Reject
- `GET /api/admin/reports` - Sales reports

### Staff (7 endpoints)
- `GET /api/staff/posted-items` - View postings
- `PUT /api/staff/posted-items/:id` - Accept/reject
- `POST /api/staff/payments` - Make payment
- `GET /api/staff/expenses` - View expenses
- `POST /api/staff/expenses` - Add expense
- `GET /api/staff/dashboard` - Staff stats
- `GET /api/staff/notifications` - Notifications

**Total: 30+ endpoints, all documented with examples**

   - Supabase dashboard for database issues

3. **Common Issues & Fixes**
   - Port already in use → Change PORT in .env
   - Can't login → Create user in Supabase console
   - Styles not showing → Run `npm run build && npm run dev`
   - API 401 error → Check JWT token is valid
   - CORS error → Verify CORS in backend .env
